**Task types** are the labels your team chooses when scheduling work on the calendar — for example, an installation, a service call, or a follow-up visit. Setting up a clear, consistent list keeps everyone's calendar easy to read and makes it simple to tell at a glance what kind of work each scheduled task involves.

<Note>
Task types live in **Settings**, which is available to administrators. If you don't see the Settings area, you don't have access — ask an administrator on your team to make changes for you.
</Note>

## Find the task types list

<Steps>
  <Step title="Open Settings">
    Go to **Settings** from the main navigation.
  </Step>
  <Step title="Go to Calendar">
    Open the **Calendar** section of Settings.
  </Step>
  <Step title="Open the Task Types tab">
    Switch to the **Task Types** tab. You'll see the full list, with each task type's **Name**, **Description**, and the dates it was created and last updated.
  </Step>
</Steps>

<Tip>
Use the **Search by name** box at the top of the list to jump straight to a task type when the list gets long.
</Tip>

## Create a task type

<Steps>
  <Step title="Start a new task type">
    On the Task Types tab, click **Create New**. The **Create Task Type** window opens.
  </Step>
  <Step title="Give it a name">
    In the **Name** field, type what your team should call this kind of work — keep it short and recognizable, like *Installation* or *Service call*. A name is required.
  </Step>
  <Step title="Add a description (optional)">
    Use the **Description** field to add a short note about when to use this task type. This is optional, but a one-line description helps the rest of the team pick the right one.
  </Step>
  <Step title="Save it">
    Click **Create**. The new task type appears in the list right away and is ready to use when scheduling on the calendar.
  </Step>
</Steps>

## Edit a task type

<Steps>
  <Step title="Open the task type">
    Find the task type in the list and click the **pencil** icon in its row. On a phone, tap the **menu** button on the task type's card and choose **Edit**.
  </Step>
  <Step title="Make your changes">
    Update the **Name** or **Description**, then click **Save Changes**.
  </Step>
</Steps>

<Note>
Renaming a task type updates it everywhere it's already in use, so existing scheduled work keeps the new label automatically.
</Note>

## Delete a task type

<Steps>
  <Step title="Choose the task type to remove">
    Find it in the list and click the red **trash** icon in its row, or use the **menu** button on its card on a phone and choose **Delete**.
  </Step>
  <Step title="Confirm">
    A confirmation appears. Click **Delete** to remove it for good.
  </Step>
</Steps>

<Note>
Deleting a task type doesn't break any work that's already scheduled. Any existing scheduled tasks that used the deleted type are moved over to your default task type, so nothing is left without a label.
</Note>

## The default task type

Your workspace always keeps one built-in task type named **Task**. It's the safety net the system falls back to — it can't be deleted, and its name can't be changed, so there's always a valid type for scheduled work to land on. You can still give it a **Description**, and you can create as many of your own task types alongside it as you need.

## Tips for a tidy list

<Tip>
A short, consistent set of task types works best. Aim for the handful of work categories your team actually schedules, rather than a long list of one-off labels — it keeps the calendar readable and makes scheduling faster for everyone.
</Tip>
